2. Booking Flow — End-to-End Sequence

Luồng customer đặt vé hoàn chỉnh: search → hold → pay → confirm → email.
sequenceDiagram autonumber actor C as 👤 Customer participant GW as API Gateway participant KC as Keycloak participant BK as booking participant FS as flight-search participant US as user participant R as Redis participant PM as payment participant PG as Payment GW participant K as Kafka participant NT as notification participant MQ as RabbitMQ participant MH as MailHog C->>GW: POST /bookings/hold (JWT, flightId, seat) GW->>KC: validate JWT KC-->>GW: ✓ valid GW->>BK: forward BK->>FS: Feign GET seat availability FS-->>BK: available + price BK->>US: Feign GET user US-->>BK: ACTIVE BK->>R: SETNX seat:VN201:12A TTL=10m R-->>BK: OK ✓ BK->>BK: INSERT booking (HELD) + outbox BK-->>C: 201 bookingId BK->>K: publish booking.held C->>PM: POST /payments PM->>PG: charge PG-->>PM: SUCCESS ✓ PM->>K: publish payment.completed K->>BK: consume payment.completed BK->>BK: UPDATE booking=CONFIRMED BK->>R: DEL seat lock BK->>K: publish booking.confirmed K->>NT: consume booking.confirmed NT->>NT: render template NT->>MQ: cmd.email.send MQ->>NT: deliver to worker NT->>MH: SMTP send MH-->>C: 📧 email

Khi payment fail, booking tự rollback (release ghế) qua event.

5. Circuit Breaker States (Resilience4j)

Khi flight-search die, circuit OPEN tránh cascade failure.
stateDiagram-v2 [*] --> CLOSED: khởi tạo CLOSED --> OPEN: ≥50% fail
trong 10 request OPEN --> HALF_OPEN: sau 30s cooldown HALF_OPEN --> CLOSED: 3 request liên tiếp OK HALF_OPEN --> OPEN: ≥1 request fail OPEN --> OPEN: fail fast
(không gọi callee) note right of CLOSED ✅ Bình thường cho mọi request đi qua end note note right of OPEN 🔴 Service đang chết Fail fast → fallback response end note note left of HALF_OPEN 🟡 Đang test thử cho 1 vài request end note
